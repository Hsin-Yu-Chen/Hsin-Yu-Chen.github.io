Baseboard Management Controller (BMC)

  1. Service processor which can monitoring the physical state of servers, computers or other hardware devices with help of sensors.
    The IPMI specifications define standardized, abstracted interfaces to the platform management subsystem.
  2. The IPMI specifications define standardized, abstracted interfaces to the platform management subsystem. IPMI includes the definition of interfaces for extending platform management between board within the main chassis, and between multiple chassis.

Inter-Integrated Circuit (I²C)

  1. Uses only two bidirectional open-collector or open-drain lines: serial data line (SDA) and serial clock line (SCL), pulled up with resistors.
  2. A master/slave architecture, which means that there can usually only be one master in a system, and other devices can only be slaves.
  3. Has a 7-bit address space, with a rarely used 10-bit extension.
  4. Mainly used as communication between ICs.
  5. Suitable for setting the initial value of IC, or for the transmission of control signals between ICs.

System Management Bus (SMBus)

  1. Derived from I²C.
  2. Consists of two lines, one is the DATA line (SDA) and the other is the CLOCK line (SCL).
  3. Collect the information of the devices on the system, such as battery capacity, CPU temperature, etc., so that the system monitor can notify the operating system to do power management processing according to the obtained information.
  4. Used to communicate with ME.

Power Management Bus (PMBus)

  1. It is an extension of SMBus.
  2. Mainly used for communication between power converters or other devices.
  3. Used to communicate with Power Supply.

Platform Environment Control Interface (PECI)

  1. PECI offers an interface for thermal monitoring of Intel processor and chipsetcomponents.
  2. A single-wire serial bus is used to support DTS (Digital Thermal Sensor).
  3. DTS is an on-chip temperature sensor with an analog-to-digital temperature converter.
  4. When DTS senses and stores the result in CPU register (MSR), PECI can obtain data from DTS; in other cases, it can only obtain data directly through BIOS.
  5. PECI’s temperature report from DTS is negative.
  6. In-Band can read the DTS temperature of the CPU core through the MSR register.
  7. Read the temperature of the CPU core DTS through PECI to control the CPU fan or read other sensors (such as the SMART information of the disk, the temperature sensor information of the DIMM, etc.) to provide the temperature value to the internal fan control system.
  8. There are two ways to read CPU temperature :
    (1) Obtain temperature data directly from the CPU through the PECI bus.
    (2) Obtain the CPU core temperature from the ME on the South Bridge through the IPMI protocol, and the ME needs to obtain the temperature from the CPU through the PECI interface.

General Purpose Input Output (GPIO)

  1. GPIO pins can be configured to be input(GPI) or output(GPO).
  2. GPIO pins can be enabled/disabled.
  3. Input values are readable (usually high or low).
  4. Output values are writable/readable.
  5. Input values can often be used as IRQs (usually for wakeup events).
